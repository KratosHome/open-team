# OpenTeam Project Core

OpenTeam is a platform for project creation and learning, where users can share ideas, build teams, and execute projects across various domains (Gaming, Business, Social, etc.).

## Core Philosophy
The platform combines project management (Jira-style) with a token-based economy to incentivize quality work and long-term sustainability.

## Key Features

### 1. Project Lifecycle
- **Idea Submission**: Users can post project ideas to attract interest.
- **Team Building**: Ideas can gather teams of specialists.
- **Execution**: Projects are managed using an integrated task manager (similar to Jira).

### 2. Tokenomics & Incentives
- **Fixed Supply**: Each project has a limited number of tokens allocated to it.
- **Task Rewards**: Team members receive tokens for every successfully completed task.
- **Marketing Incentives**: A portion of project tokens is reserved for advertising. Marketers/promoters receive tokens for successful outcomes (e.g., closed tasks related to growth).
- **Sustainability Model**: Successful projects contribute **20% of their revenue/value** back to support the ecosystem token.

## Project Domains
OpenTeam is agnostic to the sphere of work. Projects can be:
- **Game Development**
- **Business Ventures**
- **Social & Community Projects**
- **Educational Initiatives**

## Guidelines for AI Assistant
When assisting with OpenTeam:
1. **Consistency**: Ensure all features being developed align with the token-based reward system.
2. **Context**: Remember that project management is central (tasks, teams, progress).
3. **Economics**: Keep the "20% support fee" and "limited token supply" rules in mind when designing backend or economic models.
